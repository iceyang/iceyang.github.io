配置反向代理

Nginx代理websoket，与普通配置基本相同，但是需要设置好头部信息：

  • Host
  • Upgrade
  • Connection
    这几个字段表示服务器请求升级为websocket协议。

这个时候，假如我们在location之内，重新设置了一次Host，则Upgrade与Connection会失效，导致websocket连接不成功。所以如果需要重新设置Host，要确保另外两个头部同时会重新设置。
